17  // implementing class will need to provide to make the Peripheral feature
18  // complete. The number of virtual methods should be as low as possible to

22  // By using virtual, the platform implementation can override these methods

24  // These methods are pure (= 0) to force the implementing class to
25  // implement every single virtual method specified here.
26 
27  // 4. Class must be operable when made const (Required)
28  // every method must be made const, such that the object itself
29  // can be used in a const context.

31  // 5. Refrain from interface constructors (Best Practice)
32  // Interfaces should refrain from having their own constructor.
33 
34  // 6. Every peripheral must have an Initialize() method (Required)
35  // Every peripheral must have an Initialize() method that the user can use to
36  // turn on and enable the hardware. This is the first method the user must
37  // execute before running any other methods of the peripheral. Initialize
38  // should return a status indicating if it was successful, or if some error
39  // occurred.
40  virtual Status Initialize() const = 0;
41  // 7. API for data transfer
42  // If your peripheral is a communication protocol, the standard for sending
43  // and receiving data is to follow the method signature. A write or read that
44  // only transfer a single byte is not optimal as a function call is not
45  // optimal when stream large amounts of data.
46  virtual bool Write(const uint8_t * data, size_t length) const = 0;
47  // 7.1 Read Timeouts
48  // Timeout should not be apart of the API if it doesn't make sense for the
49  // protocol. Example would be something like SPI where the master drives the
50  // read operation. This is useful for something like UART where the event of
51  // receiving data is asynchronous.
52  virtual Status Read(uint8_t * data,
53  size_t length,
54  uint32_t timeout) const = 0;
55  // 8. Setters
56  // Should be prefixed with the word "Set". Setters are good for
57  // configuring the peripheral after Initialize() has run.
58  // Here are a few examples.
59  virtual void SetValue(uint32_t value) const = 0;
60  virtual void SetClockRate(uint32_t frequency) const = 0;
61  // 9. Getters
62  // Should be prefixed with the word "Get". Getters should retrieve
63  // the current configuration of the peripheral. These are generally
64  // discouraged as adding them increases the vtable without bring much value.
65  // Rather than supply Getters, unit tests should make sure that the setters
